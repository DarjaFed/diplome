Дипломный проект: Автоматизация тестирования веб-приложения покупки тура
Описание проекта

В рамках дипломного проекта реализовано автоматизированное тестирование веб-приложения покупки туристического тура.

Приложение поддерживает два способа оплаты:

покупка тура по банковской карте
покупка тура в кредит

Цель проекта — автоматизировать проверку пользовательских сценариев, валидации формы оплаты и записи данных в базу данных.

Технологии, использованные в проекте

В проекте используются следующие технологии:

Java 17
JUnit 5
Selenide
Allure Report
Docker
MySQL
PostgreSQL
Gradle
Структура проекта

Проект построен по архитектуре Page Object.

Основные пакеты:

tests — тестовые классы
pages — Page Object страницы оплаты
data — генерация тестовых данных
docker-compose.yml — запуск базы данных и сервисов
Реализованные сценарии тестирования

В проекте реализовано 39 тестовых сценариев.

Тестирование покупки по карте

Проверяются:

успешная оплата
отклонённая карта
пустые поля формы
невалидный месяц
невалидный год
просроченная карта
неверный CVV
неверное имя владельца карты

Всего: 18 тестов

Тестирование покупки в кредит

Сценарии полностью повторяют тестирование покупки по карте (по требованию преподавателя), но проверяют отдельный функционал.

Всего: 21 тест

Проверка базы данных

Реализованы тесты:

проверка записи APPROVED в БД
проверка записи DECLINED в БД
Запуск проекта
1. Запуск базы данных и сервисов
   docker-compose up -d
2. Запуск тестов
   ./gradlew clean test
3. Формирование отчёта Allure
   ./gradlew allureServe

После выполнения команды откроется отчёт в браузере.

Что проверяют автотесты

Автотесты проверяют:

корректность работы формы оплаты
валидацию пользовательских данных
обработку успешных и неуспешных платежей
корректную запись данных в базу данных
работу покупки тура в кредит
