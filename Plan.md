# План автоматизации тестирования
## 1. Перечень автоматизируемых сценариев

В рамках дипломного проекта будет автоматизировано тестирование веб-приложения интернет-магазина (aqa-shop.jar) с использованием симулятора оплаты (gate-simulator) и базы данных PostgreSQL.

### 1.1 Основные пользовательские сценарии

Будут автоматизированы следующие пользовательские сценарии:

* Открытие главной страницы интернет-магазина
* Проверка загрузки страницы оплаты
* Покупка товара по карте со статусом APPROVED
* Покупка товара по карте со статусом DECLINED
* Проверка отображения сообщений об успешной оплате
* Проверка отображения сообщений об ошибке оплаты

### 1.2 Проверка формы оплаты

Будут автоматизированы проверки формы оплаты:

* ввод валидного номера карты
* ввод невалидного номера карты
* проверка пустых полей
* проверка обязательных полей
* проверка формата месяца
* проверка формата года
* проверка формата CVV
* проверка имени владельца карты

### 1.3 Негативные сценарии

Будут автоматизированы негативные тесты:

* оплата с пустыми полями
* оплата с неверным номером карты
* оплата с истёкшим сроком карты
* оплата с некорректным CVV
* оплата с неправильным форматом имени

### 1.4 Проверка базы данных

Будет автоматизирована проверка данных в базе PostgreSQL:

* проверка записи успешной оплаты
* проверка записи отклонённой оплаты
* проверка статуса транзакции в базе
* проверка корректности данных заказа
